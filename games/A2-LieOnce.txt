LieOnce is a 2-player number-guessing game where each player is allowed to lie exactly once during the entire game.

## Game Overview

Each player secretly picks an integer between 1 and 100 (inclusive). Picks are made simultaneously and remain locked for the entire game. Players then alternate turns: on your turn, you guess the opponent's number; the opponent must respond with "Up", "Down", or "Correct".

The first player to make a guess that is BOTH genuinely correct AND confirmed as "Correct" by the opponent wins.

The catch: each player has the right to lie exactly once during the entire game, but only within the first 10 responses they give. The lie can be used on any single response among your first 10 (truth was "Up" but you say "Down", truth was "Correct" but you say "Up", truth was "Down" but you say "Correct", etc.). Using the lie is optional; not using it is also a valid strategy. Lying twice, or lying on your 11th-or-later response, is forbidden and results in immediate forfeiture.

## Agent Interface

Implement a class named `LieOnceAgent` with three methods:

```python
class LieOnceAgent:
    def __init__(self, name: str):
        """Initialize the agent. A fresh instance is created for every game."""
        self.name = name

    def pick_number(self) -> int:
        """
        Called once at the start of the game. Return an integer in [1, 100].
        Returning a non-integer or out-of-range value causes immediate game loss.
        """
        return 50

    def guess(self, state: dict) -> int:
        """
        Called when it's your turn to guess the opponent's number.
        Return an integer.
        - Non-integer returns lose your turn (no feedback exchanged, opponent plays next).
        - Out-of-range integers (e.g. 0, 101, -5) are legal but can never match a valid pick.
        """
        return 50

    def respond(self, state: dict, opponent_guess: int) -> str:
        """
        Called when the opponent has just guessed your number.
        Return one of "Up", "Down", or "Correct".
        - "Up"      means your number is HIGHER than opponent_guess.
        - "Down"    means your number is LOWER than opponent_guess.
        - "Correct" means opponent_guess equals your number.
        You may return a response that contradicts the truth (a lie) at most ONCE per game.
        Returning anything else, or lying a second time, causes immediate game loss.
        """
        return "Correct"
```

## State Dictionary

Both `guess()` and `respond()` receive a `state` dict with these keys:

```python
{
    "your_number": int,          # The number you picked at the start of the game.
    "your_lie_used": bool,       # True if you have already used your lie this game.
    "your_response_count": int,  # Number of responses YOU have already given so far this
                                 # game (0 before your first respond call). The lie may
                                 # only be used while this value is < 10. On respond calls
                                 # where this value is >= 10, lying causes forfeit.
    "turn": int,                 # Current turn number (1-indexed, combined across both players).
    "you_started": bool,         # True if you guessed first this game.
    "is_tie_attempt": bool,      # True only when this is the non-starter's tie-attempt turn.
    "last_response": str|None,   # The opponent's response ("Up"/"Down"/"Correct") to YOUR
                                 # most recent guess. None on your very first guess. The value
                                 # may be a lie — you only know the opponent has lied at most once.
}
```

You see ONLY your own state. You do NOT see the opponent's lie status, their guesses' truthfulness, or any other internal info. The runner does NOT replay past guesses or feedback to you on each call: you are responsible for tracking your own history of guesses you made, feedback you received, opponent guesses you saw, and responses you gave.

## Game Flow

1. Both agents are constructed and `pick_number()` is called on each. Picks are locked.
2. The "starter" is determined by game number within the match: Agent-1 starts game 1, Agent-2 starts game 2, alternating thereafter.
3. The starter calls `guess()`. The opponent calls `respond(state, opponent_guess)`. Then the opponent guesses; the starter responds. Play alternates.
4. The game ends when one of:
   - A guess is genuinely correct AND the responder says "Correct" (the guesser wins, subject to a possible tie-attempt — see below).
   - A player commits a forfeit-level error (see Error Handling).
   - 100 total turns elapse with no winner (DRAW).

## Lying Rules

- A "lie" is any `respond()` return that does not match the truth.
- Each agent may lie AT MOST ONCE per game. Attempting a second lie causes immediate forfeit.
- The lie window is restricted to your FIRST 10 responses. If you attempt a lie on your 11th-or-later response (i.e. when `your_response_count >= 10` at the time of the call), it is treated as an immediate forfeit, even if your lie is still unused. This rule applies equally during the tie-attempt: if the starter has already given 10 or more responses, their tie-attempt response cannot be a lie.
- Possible lies include:
  - Saying "Up" or "Down" when the truth is "Correct" (i.e. the opponent guessed your number, but you do not confirm). This consumes your lie and the game continues — the opponent will not know they were correct.
  - Saying "Correct" when the truth is "Up" or "Down" (claiming the opponent guessed correctly when they didn't). This consumes your lie and the game continues — the runner will NOT end the game because your "Correct" was false.
  - Saying "Up" when truth is "Down" (or vice versa).
- The runner verifies every response against the truth and tracks lie counts internally. There is no way to hide a lie from the runner.

## Winning a Guess

The runner ends the game ONLY when a guess is genuinely correct AND the response is "Correct". If the truth is "Correct" but the responder lies and says "Up" or "Down", the lie is consumed and play continues — the guesser does not learn they were correct (they will see "Up" or "Down" feedback like normal).

If a responder falsely says "Correct" when the guess was wrong, the lie is consumed and play continues. On the next turn the guesser will be asked to guess again, which signals to them that the previous "Correct" was a lie and the responder's lie has now been spent.

## Tie-Attempt Rule

If the STARTER wins (their guess is confirmed "Correct" truthfully), the non-starter is given ONE final guess (the "tie-attempt") to even the score, since the starter had a +1 turn advantage. The starter then responds to this final guess.

Outcomes of the tie-attempt:
- Non-starter's tie-guess is genuinely correct AND starter says "Correct" truthfully → DRAW.
- Non-starter's tie-guess is genuinely correct AND starter lies (says "Up"/"Down"):
  - If starter has not used their lie → lie consumed, starter wins (the lie acts as a "joker" to deny the tie).
  - If starter has already used their lie → second-lie forfeit, non-starter wins.
- Non-starter's tie-guess is wrong → starter wins (regardless of what feedback starter gives, unless starter forfeits via invalid response or second lie).
- Non-starter's tie-guess is non-integer or crashes → starter wins (no respond call is made).

If the non-starter wins (their guess is confirmed "Correct" truthfully) before the starter does, no tie-attempt is given (the starter already had their turn advantage and used it).

## Total Turn Cap

A "turn" = one guess by one agent (a guess + the opponent's response together count as a single turn). The cap is 100 total turns combined. If neither agent wins within 100 turns, the game is a DRAW.

A non-integer guess still consumes a turn (no feedback exchanged, play passes to opponent).

The tie-attempt counts as a normal turn (turn N+1 where the starter won on turn N).

## Scoring

- **Win:** 3 points; tiebreaker score = `100 - winner_guess_count`, where `winner_guess_count` is the number of guesses the winner made (including the winning guess).
- **Loss:** 0 points; tiebreaker score = `-(100 - winner_guess_count)`.
- **Draw (turn cap or tie-attempt success):** 1 point each, score 0 each.

A faster win produces a higher score. A loss by forfeit on the very first opportunity (e.g. invalid pick) gives the opponent the maximum score of 100.

## Error Handling

| Failure Type                                    | Consequence                                                                                  |
| ----------------------------------------------- | -------------------------------------------------------------------------------------------- |
| Initialization crash                            | That agent loses immediately (opponent wins, score = 100).                                   |
| `pick_number()` non-integer / out-of-range / crash / timeout | That agent loses immediately (opponent wins, score = 100).                                   |
| `guess()` non-integer / crash / timeout         | Lose your turn (turn count still increments). No feedback exchanged. Play passes to opponent.|
| `guess()` out-of-range integer                  | LEGAL. Opponent responds normally. (Such a guess can never be "Correct" against a valid pick.)|
| `respond()` returns an invalid string           | That agent loses immediately. Opponent wins as if they had guessed correctly on this turn.    |
| `respond()` crash / timeout                     | That agent loses immediately. Opponent wins as if they had guessed correctly on this turn.    |
| `respond()` second lie                          | That agent loses immediately. Opponent wins as if they had guessed correctly on this turn.    |
| `respond()` lie on response 11 or later         | That agent loses immediately. Opponent wins as if they had guessed correctly on this turn.    |

The 1-second move time limit (`signal.alarm`) applies to `pick_number()`, `guess()`, and `respond()`.

## Competition Format

- **1 vs 1 Match**: Two agents play `NUM_OF_GAMES_IN_A_MATCH` (default 100) games. The starter alternates each game, so over an even number of games each agent starts equally often.

### Match Points (Primary Metric)
- **Win:** 3 points
- **Draw:** 1 point each
- **Loss:** 0 points

### Tie-Breaker Score (Secondary Metric)
- **Win:** `100 - winner_guess_count` (faster win → higher score; max 99 for a 1-guess win, max 100 for a forfeit-before-any-guess).
- **Loss:** the negative of the winner's score.
- **Draw:** 0 for both players.
